NAME

asyncy — Write stories, then code.

DESCRIPTION

Welcome to Asyncy Alpha Release! We are proud and humbled to have you join our journey.

Your feedback is very important and we value the good, the bad and the ugly.

The Alpha environment is single-application with many caveats.

USAGE

During Alpha you can write stories, build services and run application, almost like you would in Asyncy Beta. We found the best way to build an application with Asyncy is (1) start with the goal of the application, (2) write stories with assumptions, (3) discover and build services.

asyncy build — We only support syntax debugging at this time which will output debug information about your application.

asyncy deploy OR git push asyncy master — Will deploy your application to your locally running Asyncy environment. Make sure you commit changes first.

CAVEATS

Single Mode Asyncy Alpha is single application mode and can only run one-app per machine, so jam it all in one repo!

Devtools are not built yet. We plan to build IDE plugins, syntax highlighting, autocompleting and more which will significantly help writing stories and debugging.

Service constraints. Currently there is no support for containers that stream information (such as streaming tweets into Storyscript).

Microservices will not scale. During Alpha serices do not scale and are 1 container per service. It wont be an issue, but just to keep in mind.
